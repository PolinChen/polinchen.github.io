Log集中管理

日誌分析是排除系統故障最常用的手段。通過BiMAP 維運分析系統收集核心應用運行過程中所產生的日誌，包含主機伺服器、存儲系統、資料庫、網路、應用軟體等日誌。通過這些數量巨大、結構複雜、類型眾多Log的集中管理，除了協助系統維運以及除錯外，更可以提供 IT 人員未來執行系統優化和擴容計畫時的確切可行方向。


Log檢索分析

利用BigData 分析技術，將集中的所有資料屬性、內容迥異的結構化和非結構化log，進行資料梳理和整合，轉換為統一格式的資料，並進行高速的全文檢索、彈性的關鍵字搜索功能、過濾、統計、可視化等分析工作，使得日誌數據的價值得到最充分的挖掘。

Log可視化

BiMAP 採用 ELK stack 的 Kibana 動態儀表板設計，可以自定義多種表格、柱狀圖、餅狀圖、折線圖等不同的可視化介面，以對存儲在Elasticsearch中的數據進行深入挖掘分析與視覺化，更能幫助運維團隊提升了工作效率。








應用Log 集中分析技術，提昇iT 維運品質

通過BiMAP 對主機伺服器、存儲系統、資料庫、網路、應用軟體等日誌的集中收集、處理和預測分析；幫助你快速解決故障，並排查出運行過程中的高風險點和弱點，為運維決策的預測依據，在異常現象出現之前，採取有效的防範措施和手段，以減少或消除可能發生的事故， 進而發揮大數據對運維決策分析和防患於未然的重要作用！

軟體品質的量測和驗證

良好的軟體品質管理主要是運用品管的觀念與技術，對於軟體專案進行過程的軟體品質作一系列完整的規劃、執行與控制的活動。 其主要目的是提升軟體品質水準與設計生產力。

優良的軟體品質源於良好的用戶體驗，因為軟體質量問題如：響應時間太長、軟體報錯、無法正常連結，甚至是當機的現象，造成用戶不同程度的抱怨的用戶體驗

BiMAP 整合系統和軟體的日誌，量化軟體品質的指標，並且進行關聯量測、驗證和確認。並且提供給開發團隊和測試部門便利的監控和分析儀表版，以將軟體的缺點與軟體專案失敗的風險減少至最低程度。

提供升級改造的量化依據，保障投資效益

隨著核心應用系統複雜度和的資料量日益增長，IT系統也會隨需而變。如何利用大數據維運分析平台，如何量化評估應用系統的負載需求和資源設備的最大業務支撐力，來保障核心系統未來3-5年的高效穩定運行。避免升級方案實施後，才發現改善效果不如預期的現象。

建制預警通報機制，落實主動維運模式

通過現有收集的效能 log， 經由資料挖掘、建模分析，建立系統之間的關聯性，建立IT系統的風險相關模型，以大數據技術實現風險預警通報管理機制， 一方面提供技術團隊的提前實施調整異常的依據，另一方面協助管理團隊建立主動維運的模式，提早維運決策品質和降低系統運營的風險。

縮短異常問題的處理週期，提升體驗滿意度

由於IT系統環境日益複雜、通常只有在核心系統發生服務中斷後，并造成影響時，IT部門才會開始被動分析異常問題。由於需要通過不同的技術和多種監控工具，對龐大的複雜資料做繁複的診斷，造成處理問題的周期過長。

DevOps 專業培訓和顧問諮詢

DevOps有效率地整合開發、交付及維運，讓這些傳統上隔離的孤島可以精實且流暢的連結在一起。 藉助 DevOps 將開發和維運合而為一，來實現敏捷高效率的自動化 IT。

提供DevOps 培訓和實作服務，讓開發、測試和維運團隊有效的利用自動化工具，並串接於不同的職能團隊之間，以便持續提供客戶服務，包含持續整合(continuous integration)、 持續部署 (continous deployment)」、 與持續交付 (continous delivery)。

DevOps 中的自動化部署，即是利用工具做自動化部署，減少手動以及傳遞的時間，且可以避免人為錯誤，改善軟體交付品質。 依據測試、驗收或上線實際需求，快速自動部署到不同的環境，常見的有：整合測試環境 (System Integration Tests)、測試環境 (QA )、用戶驗收測試環境 (UAT)、上架環境 (Staging)、以及生產環境 (Production)。
